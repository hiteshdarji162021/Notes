## LLM — Legacy Completions

The base `OpenAI` class calls the older **completions** endpoint (`/v1/completions`). You pass a raw string and get a raw string back.

> ⚠️ **Deprecated pattern.** Only `gpt-3.5-turbo-instruct` still supports this endpoint. All GPT-4 class models require the chat interface. Prefer `ChatOpenAI` for all new projects.

```python
from langchain_openai import OpenAI
from dotenv import load_dotenv

load_dotenv()

llm = OpenAI(model='gpt-3.5-turbo-instruct')
result = llm.invoke('what is the capital of India')
print(result)
# Output: raw string e.g. "\nNew Delhi is the capital of India."
```

**Key behaviour**
- Input → raw `str`
- Output → raw `str`
- No system prompt or multi-turn support
- Endpoint: `/v1/completions`

## ChatModel — Modern Interface

`ChatOpenAI` uses the **chat completions** endpoint (`/v1/chat/completions`). It accepts message objects and returns an `AIMessage`. Passing a plain string to `.invoke()` is automatically wrapped in a `HumanMessage`.

```python
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv

load_dotenv()

model = ChatOpenAI(
    model='gpt-4o',                # ✅ prefer gpt-4o over gpt-4 (faster + cheaper)
    temperature=1.5,
    max_completion_tokens=10       # ✅ use max_completion_tokens, not max_tokens (LangChain ≥ 0.2)
)

result = model.invoke('what is the capital of India')
print(result.content)
# Output: "New Delhi"
```

> 📝 **LangChain ≥ 0.2 note:** `max_tokens` still works but `max_completion_tokens` matches the OpenAI API spec and is preferred going forward.

**Multi-turn with message objects**

```python
from langchain_core.messages import HumanMessage, SystemMessage

messages = [
    SystemMessage(content="You are a geography expert. Answer concisely."),
    HumanMessage(content="What is the capital of India?"),
]

result = model.invoke(messages)
print(result.content)
```

## LLM vs ChatModel — Key Differences

| Feature | LLM (legacy) | ChatModel (current) |
|---|---|---|
| **Input** | Raw string | List of messages |
| **Output** | Raw string | `AIMessage` object |
| **API endpoint** | `/v1/completions` | `/v1/chat/completions` |
| **System prompt** | ❌ Not supported | ✅ Supported |
| **Multi-turn** | ❌ Not supported | ✅ Supported |
| **Supported models** | `gpt-3.5-turbo-instruct` only | All GPT-4o, GPT-4, etc. |
| **Status** | Deprecated | Active / recommended |
| **LangChain class** | `OpenAI` | `ChatOpenAI` |

## HuggingFace — Hosted Endpoint

Calls the **HuggingFace Inference API** — the model runs on HF's servers. Requires a `HUGGINGFACEHUB_API_TOKEN` in your `.env`.

```python
from langchain_huggingface import ChatHuggingFace, HuggingFaceEndpoint
from dotenv import load_dotenv

load_dotenv()

llm = HuggingFaceEndpoint(
    repo_id='TinyLlama/TinyLlama-1.1B-Chat-v1.0',
    task='text-generation',
    max_new_tokens=100,
    temperature=0.5,
)

model = ChatHuggingFace(llm=llm)
result = model.invoke('what is the capital of India')
print(result.content)
```

**Good for:** Quick prototyping, no GPU required, free tier available on HuggingFace.

## HuggingFace — Local Pipeline

Downloads model weights to disk and runs inference **locally** via `transformers` + `torch`. No API key needed after initial download; works fully offline.

```python
from langchain_huggingface import ChatHuggingFace, HuggingFacePipeline
from dotenv import load_dotenv

load_dotenv()

llm = HuggingFacePipeline.from_model_id(
    model_id='TinyLlama/TinyLlama-1.1B-Chat-v1.0',
    task='text-generation',
    pipeline_kwargs={
        'temperature': 0.5,
        'max_new_tokens': 100,
        'do_sample': True,        # required when temperature != 1.0
    }
)

model = ChatHuggingFace(llm=llm)
result = model.invoke('What is the capital of India')
print(result.content)
```

> 📝 Add `do_sample=True` to `pipeline_kwargs` whenever using a custom `temperature`, otherwise some `transformers` versions raise a warning or error.

---

## HuggingFace Endpoint vs Pipeline

| Feature | `HuggingFaceEndpoint` | `HuggingFacePipeline` |
|---|---|---|
| **Where it runs** | HuggingFace servers | Your machine |
| **API token needed** | ✅ Yes | ❌ No (after download) |
| **Local GPU needed** | ❌ No | Recommended |
| **Offline capable** | ❌ No | ✅ Yes |
| **Latency** | Network dependent | Hardware dependent |
| **Best for** | Prototyping, no GPU | Privacy, offline, fine-tuned models |

## Embedding Models

### What & Why

An **embedding model** converts text into a dense numeric vector (e.g. 1536 floats). Texts with similar meaning produce vectors that are geometrically close together.

**Why use embeddings?**
- Semantic similarity search without calling an LLM on every query
- Core building block of **RAG** (Retrieval-Augmented Generation)
- Efficient clustering, deduplication, and classification of text
- Powers vector databases (FAISS, Chroma, Pinecone, Weaviate)

**How it fits in a RAG pipeline:**

```
Documents → embed_documents() → vectors stored in vector DB
Query     → embed_query()     → query vector
                               → similarity search → top-k docs
                               → feed docs + query to LLM → answer
```

---

### Popular Models

**Closed source (API-based)**

| Model | Provider | Dimensions | Notes |
|---|---|---|---|
| `text-embedding-3-small` | OpenAI | up to 1536 | Best cost/performance ratio |
| `text-embedding-3-large` | OpenAI | up to 3072 | Highest accuracy, supports matryoshka truncation |
| `text-embedding-ada-002` | OpenAI | 1536 | Older, no dimension control |
| `embed-v3.0` | Cohere | 1024 | Multilingual support |
| `textembedding-gecko` | Google | 768 | Available via Vertex AI |

**Open source (local/free)**

| Model | Dimensions | Notes |
|---|---|---|
| `sentence-transformers/all-MiniLM-L6-v2` | 384 | Fast, small (~90 MB), great baseline |
| `BAAI/bge-large-en-v1.5` | 1024 | High accuracy, English focused |
| `BAAI/bge-m3` | 1024 | Multilingual, best open-source option |
| `nomic-ai/nomic-embed-text-v1.5` | 768 | Long context (8192 tokens) |
| `intfloat/e5-large-v2` | 1024 | Strong retrieval performance |

### OpenAI — Single Query

Use `embed_query()` for a single string (e.g. a user's search question).

```python
from langchain_openai import OpenAIEmbeddings
from dotenv import load_dotenv

load_dotenv()

embedding = OpenAIEmbeddings(
    model='text-embedding-3-large',
    dimensions=32    # ✅ matryoshka truncation — only works with text-embedding-3-* models
)

result = embedding.embed_query('Delhi is capital of India')
print(len(result))   # → 32
print(result)        # → list of 32 floats
```

> ⚠️ The `dimensions` parameter only works with `text-embedding-3-small` and `text-embedding-3-large`. It will raise an error with `text-embedding-ada-002`.

---

### OpenAI — Multiple Documents

Use `embed_documents()` for a list of strings (your knowledge base / corpus).

```python
from langchain_openai import OpenAIEmbeddings
from dotenv import load_dotenv

load_dotenv()

embedding = OpenAIEmbeddings(
    model='text-embedding-3-large',
    dimensions=32
)

documents = [
    "Delhi is the capital of India",
    "Kolkata is the capital of West Bengal",
    "Paris is the capital of France"
]

result = embedding.embed_documents(documents)

print(len(result))      # → 3  (one vector per document)
print(len(result[0]))   # → 32 (dimensions per vector)
```

**Rule of thumb:**
- `embed_query()` → for the user's question / search input
- `embed_documents()` → for your knowledge base / corpus

### HuggingFace — Local Embeddings

Download and run embedding models locally. No API key, no cost per call.

```python
from langchain_huggingface import HuggingFaceEmbeddings

embedding = HuggingFaceEmbeddings(
    model_name='sentence-transformers/all-MiniLM-L6-v2',
    model_kwargs={'device': 'cpu'},     # use 'cuda' if GPU available
    encode_kwargs={'normalize_embeddings': True}  # normalize for cosine similarity
)

documents = [
    "Delhi is the capital of India",
    "Kolkata is the capital of West Bengal",
    "Paris is the capital of France"
]

vectors = embedding.embed_documents(documents)

print(len(vectors))      # → 3
print(len(vectors[0]))   # → 384 (MiniLM output dimension)
```

## Similarity Search — Cosine Score

The manual similarity search pattern — embed all documents, embed the query, then rank by cosine similarity. This is the conceptual foundation of every vector database.

```python
from langchain_openai import OpenAIEmbeddings
from dotenv import load_dotenv
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

load_dotenv()

embedding = OpenAIEmbeddings(
    model='text-embedding-3-small',
    dimensions=300
)

documents = [
    "Virat Kohli is an Indian cricketer known for his aggressive batting and leadership.",
    "MS Dhoni is a former Indian captain famous for his calm demeanor and finishing skills.",
    "Sachin Tendulkar, also known as the 'God of Cricket', holds many batting records.",
    "Rohit Sharma is known for his elegant batting and record-breaking double centuries.",
    "Jasprit Bumrah is an Indian fast bowler known for his unorthodox action and yorkers."
]

query = 'tell about Rohit Sharma'

# Step 1: embed corpus (do this once and cache in production)
doc_embeddings = embedding.embed_documents(documents)

# Step 2: embed query
query_embedding = embedding.embed_query(query)

# Step 3: compute similarity scores
scores = cosine_similarity([query_embedding], doc_embeddings)[0]

# Step 4: find best match
index = int(np.argmax(scores))

print('Query   :', query)
print('Matched :', documents[index])
print('Score   :', round(scores[index], 4))
print('All scores:', [round(s, 4) for s in scores])
```

**Understanding cosine similarity scores**

| Score range | Meaning |
|---|---|
| `0.90 – 1.00` | Near-identical meaning |
| `0.70 – 0.89` | Strong semantic match |
| `0.50 – 0.69` | Partial / related match |
| `0.30 – 0.49` | Weak relationship |
| `< 0.30` | Likely unrelated |

print(result.content)                            # access the text

from langchain_huggingface import ChatHuggingFace, HuggingFaceEndpoint

## Notes & Gotchas

- **`dimensions` param** only works with `text-embedding-3-small` and `text-embedding-3-large`, not with `ada-002`.
- **`do_sample=True`** is required in `HuggingFacePipeline.pipeline_kwargs` when using a non-default `temperature`.
- **Cache document embeddings** — re-embedding the same corpus on every query wastes API calls and money. Embed once, persist to a vector store.
- **`max_completion_tokens`** is the current OpenAI-aligned parameter name (LangChain ≥ 0.2). `max_tokens` still works but may be deprecated.
- **`normalize_embeddings=True`** in `HuggingFaceEmbeddings.encode_kwargs` ensures vectors are unit-normalized, making cosine similarity equivalent to dot product (faster).
