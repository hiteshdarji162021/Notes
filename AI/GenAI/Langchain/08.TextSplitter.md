# LangChain Text Splitters (LangChain 1.x Complete Guide)

## Overview

Text Splitters are one of the most important components in:

* RAG Applications
* AI Agents
* Knowledge Bases
* Chatbots
* Code Understanding Systems
* Enterprise Search

Their purpose is to break large documents into smaller chunks before creating embeddings and storing them in a vector database.

---

# CharacterTextSplitter

## Example

```python
from langchain_text_splitters import CharacterTextSplitter

splitter = CharacterTextSplitter(
    chunk_size=200,
    chunk_overlap=0,
    length_function=len,
    is_separator_regex=False
)
```

---

## Parameters

### chunk_size

Maximum characters allowed inside a chunk.

```python
chunk_size=200
```

---

### chunk_overlap

Copies some content from previous chunk into next chunk.

```python
chunk_overlap=50
```

Example:

```text
Chunk 1:
ABCDE

Chunk 2:
CDEFG
```

Overlap:

```text
CDE
```

---

### length_function

Determines how chunk size is measured.

Default:

```python
length_function=len
```

Counts characters.

---

### is_separator_regex

Determines whether separator should be treated as regex.

```python
is_separator_regex=False
```

---

## Pros

* Very simple
* Fast

## Cons

* Can create oversized chunks
* Not ideal for production RAG

Rating: 6/10

# RecursiveCharacterTextSplitter

Most commonly used splitter.

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=200,
    chunk_overlap=20
)
```

---

## How It Works

Default separator hierarchy:

```python
[
    "\n\n",
    "\n",
    " ",
    ""
]
```

Process:

```text
Paragraph
 ↓
Line
 ↓
Word
 ↓
Character
```

---

## Advantages

* Always respects chunk size
* Better retrieval quality
* Industry standard

Rating: 10/10

---

# split_text()

Input:

```python
chunks = splitter.split_text(text)
```

Output:

```python
List[str]
```

---

# split_documents()

Input:

```python
docs = loader.load()

chunks = splitter.split_documents(docs)
```

Output:

```python
List[Document]
```

Metadata remains preserved.

Example:

```python
{
    "source":"sample.pdf",
    "page":3
}
```

# SemanticChunker

## Example

```python
from langchain_experimental.text_splitter import SemanticChunker
from langchain_openai import OpenAIEmbeddings

splitter = SemanticChunker(
    OpenAIEmbeddings(),
    breakpoint_threshold_type="standard_deviation",
    breakpoint_threshold_amount=1
)
```

---

## Why Use Semantic Chunking?

Traditional chunking:

```text
Split by size
```

Semantic chunking:

```text
Split by meaning
```

Example:

```text
Paragraph 1 -> Farming

Paragraph 2 -> IPL

Paragraph 3 -> Terrorism
```

Output:

```text
Chunk 1 -> Farming

Chunk 2 -> IPL

Chunk 3 -> Terrorism
```

---

# breakpoint_threshold_type

Determines where semantic chunk boundaries occur.

---

## 1. standard_deviation

```python
breakpoint_threshold_type="standard_deviation"
```

Most commonly used.

Breaks when similarity drops significantly.

### Best For

* General RAG
* Knowledge Bases
* Production Systems

Rating: 10/10

---

## 2. percentile

```python
breakpoint_threshold_type="percentile"
```

Example:

```python
breakpoint_threshold_amount=95
```

Creates chunks at largest semantic changes.

### Best For

* Large PDFs
* Research Papers
* Enterprise Documents

Rating: 9/10

---

## 3. interquartile

```python
breakpoint_threshold_type="interquartile"
```

Uses IQR calculations.

### Best For

* Noisy Data
* OCR Documents
* Messy Content

Rating: 9/10

---

## 4. gradient

```python
breakpoint_threshold_type="gradient"
```

Uses rate of semantic change.

### Best For

* Books
* Stories
* Articles

Rating: 9/10

# Modern Token-Based Splitting

Recommended for production.

## TokenTextSplitter

```python
from langchain_text_splitters import TokenTextSplitter

splitter = TokenTextSplitter(
    chunk_size=500,
    chunk_overlap=50
)
```

---

## Tiktoken Splitter

Best choice for OpenAI models.

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter.from_tiktoken_encoder(
    chunk_size=500,
    chunk_overlap=50
)
```

Advantages:

* Token aware
* Matches LLM limits
* Better retrieval performance

Rating: 10/10
