# LangChain Tools Complete Guide

# Overview

In LangChain, a **Tool** is a function or capability that allows an LLM or Agent to interact with the outside world.

Without tools, an LLM can only generate text based on its training data.

With tools, an LLM can:

* Search the internet
* Execute shell commands
* Query databases
* Call APIs
* Perform calculations
* Access custom business logic

# 1. DuckDuckGoSearchRun

## Purpose

Perform internet searches without requiring an API key.

## Code

```python
from langchain_community.tools import DuckDuckGoSearchRun

search_tool = DuckDuckGoSearchRun()

results = search_tool.invoke(
    "Today oneday match scoreboard mens team"
)

print(results)
```

---

## How It Works

```text
User Query
      │
      ▼
DuckDuckGoSearchRun
      │
      ▼
DuckDuckGo Search Engine
      │
      ▼
Search Results
      │
      ▼
Return Text Response
```

---

## Output Example

```text
India vs Australia
Score: 278/6
Overs: 45.2
```

---

## Use Cases

### AI News Agent

```python
search_tool.invoke("Latest AI News")
```

### Stock Market Agent

```python
search_tool.invoke("Nifty 50 today")
```

### Research Agent

```python
search_tool.invoke("Top AI startups in India")
```

---

## Advantages

| Feature        | Benefit                   |
| -------------- | ------------------------- |
| No API Key     | Easy setup                |
| Real-time Data | Latest information        |
| Agent Friendly | Can be used by LLM agents |

# 3. Tool Decorator (@tool)

## Purpose

Convert a Python function into a LangChain tool.

---

## Step 1

Normal Function

```python
def multiply(a, b):
    return a * b
```

---

## Step 2

Add Type Hints

```python
def multiply(a: int, b: int) -> int:
    return a * b
```

---

## Step 3

Add Tool Decorator

```python
from langchain_core.tools import tool

@tool
def multiply(a: int, b: int) -> int:
    """Multiply two numbers"""
    return a * b
```

---

## Invocation

```python
result = multiply.invoke(
    {"a": 3, "b": 5}
)
```

---

## Output

```text
15
```

---

## Tool Metadata

```python
print(multiply.name)
print(multiply.description)
print(multiply.args)
```

Output:

```text
multiply
Multiply two numbers
{
  a:int,
  b:int
}
```

---

## Architecture

```text
Python Function
      │
      ▼
@tool
      │
      ▼
LangChain Tool
      │
      ▼
Agent Can Use It
```

---

## Best Use Case

Simple business logic.

Examples:

* Add numbers
* Send email
* Validate user
* Generate report

# 4. StructuredTool

## Purpose

Create tools with strict input validation.

---

## Why StructuredTool?

Suppose a tool requires:

* Name
* Email
* Age

Validation becomes important.

---

## Code

```python
from langchain_core.tools import StructuredTool
from pydantic import BaseModel, Field

class MultiplyInput(BaseModel):
    a: int = Field(...)
    b: int = Field(...)

def multiply_func(a: int, b: int):
    return a * b
```

---

## Create Tool

```python
multiply_tool =
StructuredTool.from_function(
    func=multiply_func,
    name="multiply",
    description="Multiply two numbers",
    args_schema=MultiplyInput
)
```

---

## Invoke

```python
multiply_tool.invoke({
    "a": 3,
    "b": 3
})
```

---

## Output

```text
9
```

---

## Validation Example

Wrong Input:

```python
{
  "a": "abc",
  "b": 3
}
```

Pydantic Error:

```text
ValidationError
```

---

## Architecture

```text
User Input
     │
     ▼
Pydantic Schema
     │
     ▼
Validation
     │
     ▼
Function Execution
```

---

## Use Cases

### API Agents

Validate request payload.

### Financial Agents

Validate amounts.

### Medical Agents

Validate patient data.

# 5. BaseTool

## Purpose

Maximum customization and control.

Used when @tool and StructuredTool are insufficient.

---

## Code

```python
from langchain.tools import BaseTool
```

---

## Input Schema

```python
class MultiplyInput(BaseModel):
    a: int = Field(...)
    b: int = Field(...)
```

---

## Custom Tool

```python
class MultiplyTool(BaseTool):

    name = "multiply"

    description = (
        "Multiply two numbers"
    )

    args_schema = MultiplyInput

    def _run(
        self,
        a: int,
        b: int
    ) -> int:
        return a * b
```

---

## Invoke

```python
multiply_tool.invoke({
    "a": 3,
    "b": 3
})
```

---

## Output

```text
9
```

---

## Architecture

```text
BaseTool
    │
    ├── Name
    ├── Description
    ├── Schema
    └── _run()
```

---

## When To Use BaseTool

Use when:

* Authentication required
* API calls required
* Logging required
* Retry logic required
* Complex business workflows

---

## Real Example



# 6. Toolkit

## Purpose

Group multiple related tools together.

---

## Tools

```python
@tool
def add(a:int,b:int):
    return a+b

@tool
def multiply(a:int,b:int):
    return a*b
```

---

## Toolkit

```python
class MathToolkit:

    def get_tools(self):
        return [
            add,
            multiply
        ]
```

---

## Usage

```python
toolkit = MathToolkit()

tools = toolkit.get_tools()
```

---

## Output

```text
add => Add two numbers
multiply => Multiply two numbers
```

---

## Architecture

```text
Toolkit
 │
 ├── Tool 1
 │
 ├── Tool 2
 │
 ├── Tool 3

 └── Tool N
```

# Interview Questions

### What is a Tool?

A capability that allows an LLM to perform actions outside its own reasoning.

### Difference Between Tool and Agent?

Tool performs an action.

Agent decides which tool to use.

### Difference Between @tool and StructuredTool?

@tool is simple.

StructuredTool provides schema validation.

### Difference Between StructuredTool and BaseTool?

StructuredTool wraps a function.

BaseTool provides full customization.

### Why Use Toolkit?

To organize related tools into a reusable collection.
