## PART 1Ô∏è‚É£ Other (Custom) Dropdowns (Non-`<select>`)

### What is an "Other Dropdown"?
- Built using `div`, `span`, `ul`, `li`
- NOT a `<select>` tag
- Common in modern UI frameworks

---

## Example: Custom Dropdown Selection

### Code
```ts
test('Other dropdown', async ({ page }) => {
  await page.goto('https://naveenautomationlabs.com/opencart/ui/dropdowns.html');

  await selectDropdown(page, 'Choose your preferred programming language', 'Java');
  await selectDropdown(page, 'Choose your preferred web framework', 'Angular');
  await selectDropdown(page, 'Select your experience level', 'Mid-level (4-6 years)');

  await page.pause();
});

async function selectDropdown(page: Page, dropdown: string, value: string) {
  await page.locator(`//div[@class='select-trigger']//span[text()='${dropdown}']`).click();
  await page.getByText(value, { exact: true }).click();
}
```

### Output
- Selected values are displayed in UI

### Use case
- React / Angular / Vue dropdowns
- Modern UI components

### Best practice
- Use visible text
- Wrap logic in reusable helper

## PART 2Ô∏è‚É£ Normal Frame (Single iframe)

### What is a Frame?
- An HTML `<iframe>` that loads another page inside the current page

---

## Example: Normal Frame Interaction

### Code
```ts
await page.goto('https://www.londonfreelance.org/courses/frames/index.html');

const frame = page.frameLocator("[name='main']");
const header = await frame.locator('h2').innerText();
console.log(header);
```

### Output
```text
Title Bar Frames Example
```

### Use case
- Legacy applications
- Embedded content (payment, ads)

---

## PART 3Ô∏è‚É£ Get All Frames on Page

### Code
```ts
const allFrames = await page.locator('//frame').all();
console.log('total number of frames:', allFrames.length);

for (const f of allFrames) {
  console.log(await f.getAttribute('name'), await f.getAttribute('src'));
}

console.log(page.frames().length);
```

### Output
```text
total number of frames: 3
main : main.html
menu : menu.html
top : top.html
3
```

### Use case
- Debugging frame-heavy pages
- Frame validation

## PART 4Ô∏è‚É£ Nested Frames (Frame inside Frame)

### What is a Nested Frame?
- One iframe loaded inside another iframe

---

## Example: Nested Frame Handling

### Code
```ts
test('frame and nestedframe', async ({ page }) => {
  await page.goto('https://selectorshub.com/iframe-scenario/');

  const frame1 = page.frameLocator('#pact1').first();
  const frame2 = frame1.frameLocator('#pact2');
  const frame3 = frame2.frameLocator('#pact3');

  await frame1.getByPlaceholder('First Crush').fill('abc');
  await frame2.getByPlaceholder('Current Crush Name').fill('def');
  await frame3.getByPlaceholder('Destiny').fill('ghi');

  console.log(await page.title());
  console.log(page.frames().length);

  await page.pause();
});
```

### Output
```text
SelectorsHub
3
```

### Use case
- Complex embedded systems
- Multi-layer authentication flows

## PART 5Ô∏è‚É£ `frameLocator()` vs `page.frames()`

### `frameLocator()`
- Used to **interact with elements** inside frames
- Auto-wait supported

### `page.frames()`
- Returns all frames as `Frame[]`
- Mostly used for **debugging & inspection**

---

## ‚úÖ Best Practices (IMPORTANT)

### Dropdowns
- Use text-based selection
- Avoid index-based logic
- Create reusable helpers

### Frames
- Always identify frame depth first
- Use CSS to locate iframe
- Interact from the deepest frame
- Prefer `frameLocator()` over `frame()`

---
